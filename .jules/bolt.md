## 2024-04-16 - [Optimize Alert Throttling in Detection Loop]
**Learning:** Found a performance bottleneck in `fastapi_server.py` where `datetime.strptime()` was being used inside the `detection_loop` (a ~30 FPS vision hot loop) to parse the timestamp of the last alert string back into a `datetime` object, purely to check a 2-second cooldown. String formatting (`datetime.now().strftime()`) was also being executed unconditionally before the rate-limit check. This is an anti-pattern as string parsing and formatting are computationally expensive for high-frequency loops.
**Action:** Always use native float timestamps (`time.time()`) for tracking rate limits or cooldowns in hot loops. Defer expensive string formatting (like `strftime`) until *after* the condition or rate limit has passed.
