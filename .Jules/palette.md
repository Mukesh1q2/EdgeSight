## 2024-04-18 - Manage Screen Reader Focus and Politeness
**Learning:** Real-time dashboards present specific challenges for screen readers. High-frequency updates (like 10Hz telemetry charts or gauges) overwhelm screen readers if marked as live regions. Instead, only low-frequency, significant state changes (like alert logs and risk badges) should be announced, utilizing `aria-live="polite"`. Additionally, decorative icons embedded in text-labeled buttons will be double-read or read confusingly by screen readers unless explicitly hidden using `aria-hidden="true"`.
**Action:** When implementing real-time UI components, restrict `aria-live` regions to low-frequency, critical updates. For UI elements containing both icons and text, always apply `aria-hidden="true"` to the icon elements. Ensure that all interactive elements (buttons, sliders, selects) implement a distinct `:focus-visible` outline to guarantee they are obvious during keyboard navigation.
