## 2025-04-10 - Keyboard and Screen Reader Accessibility in Real-time Dashboard
**Learning:** High-frequency, real-time dashboards present unique accessibility challenges. Custom UI controls (buttons, sliders, selects) often reset or hide default focus rings, making keyboard navigation difficult. Furthermore, screen readers can be overwhelmed by high-frequency telemetry updates (like 10Hz charts or gauges) if naive `aria-live` is applied globally.
**Action:** Always manually define explicit `:focus-visible` states with offset for custom controls to guarantee keyboard visibility. Apply `aria-live="polite"` *only* to low-frequency, high-value components (like alert logs or aggregate risk badges) to ensure critical state changes are announced without spamming the user. Also, explicitly hide decorative icons within interactive elements using `aria-hidden="true"` to prevent redundant screen reader announcements.
