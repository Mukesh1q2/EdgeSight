## 2024-05-24 - Interactive Elements Focus and Live Region Attributes
**Learning:** High-frequency visual updates (e.g., telemetry 10Hz) should NOT have `aria-live` applied to prevent overwhelming screen readers. However, low-frequency dynamic components like alert logs and risk level badges critically require `aria-live="polite"` so state changes are properly announced. Additionally, interactive UI elements like buttons, select menus, and sliders need proper `:focus-visible` styling with explicit outlines to support keyboard navigation.
**Action:** Always ensure critical, low-frequency dynamic UI elements implement `aria-live` attributes. Ensure that all interactive elements across the application support keyboard accessibility by declaring a consistent `:focus-visible` style using the design system's primary colors.
