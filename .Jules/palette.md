## 2024-05-02 - Ensure Low-Frequency Alerts and Keyboard Accessibility Focus Styles
**Learning:** For dynamic real-time dashboards like EdgeSight, low-frequency updates like Risk Level changes or Alert Logs require `aria-live="polite"` to be announced by screen readers without being overwhelming, whereas high-frequency updates (like FPS/Latency) should not use it. Additionally, custom components like styled range sliders and select dropdowns must implement `:focus-visible` outlines using design tokens (like `--primary`) to preserve keyboard navigation accessibility.
**Action:** When working on real-time dashboards or custom interactive elements, always apply `aria-live` strictly for meaningful low-frequency changes, use `aria-hidden="true"` on decorative icons inside text buttons, and ensure `:focus-visible` with a 2px outline is present for all interactive elements to support keyboard users.
